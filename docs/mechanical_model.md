# Ball-and-Beam 机械结构与初步模型

本文记录车载 Ball-and-Beam 装置的机械结构、初步测量和建模边界。当前没有下位机，舵机由人工拨动；本文模型只能用于理解趋势和规划标定，不能视为已确认的控制对象模型。

## 1. 状态分类

为避免把粗测结果误当成精确参数，本文统一使用以下分类：

- **已确认结构**：由实物观察可以直接确认，但不代表尺寸已经精密测量。
- **粗略测量**：已经使用尺子或量角器测量，数值包含明显人工误差。
- **推测参数**：根据结构和现象推测，尺寸定义或测量基准尚未统一。
- **近似模型**：为了解释趋势建立的简化公式，不代表真实机构严格服从该公式。
- **尚未验证假设**：必须通过后续标定实验才能确认。

## 2. 已确认结构

- 装置将安装在红外循迹小车顶部。
- 亚克力顶板约为 `28.5 cm × 18.5 cm`。
- 白色轨道为半圆筒形，钢珠只能沿轨道长度方向滚动。
- 轨道总长度约为 `26 cm`。
- 轨道右端为固定转轴。
- 轨道左端由舵机摇臂和连杆驱动。
- K230 后续负责检测钢珠位置并通过 UART 输出测量值。
- 下位机未来负责舵机控制；当前阶段不实现闭环控制。

## 3. 粗略测量数据

下列参数来自当前实物测量，尚未经过标准化标定：

| 参数 | 暂定值 | 状态与说明 |
| --- | ---: | --- |
| 亚克力顶板 | `28.5 × 18.5 cm` | 粗略测量 |
| 轨道总长度 `L_track` | `约 26 cm` | 粗略测量 |
| 右侧固定转轴至左侧驱动点 `L_drive` | `约 24.8 cm` | 粗略测量，需确认两端基准点 |
| 舵机摇臂有效半径 `r_servo` | `约 1.5 cm` | 粗略测量，需确认实际使用孔位中心距 |
| 固定端距底板高度 | `约 6.5 cm` | 粗略测量 |
| 舵机机械活动范围 | `约 ±90°` | 人工拨动观察，不等于可安全命令范围 |
| 轨道最大倾角 | `约 -4° 至 +3.8°` | 普通量角器粗测，零位和符号仍需统一 |

当前正方向粗测趋势：

| 舵机相对中位角 α | 轨道角 θ（粗测） |
| ---: | ---: |
| `0°` | `约 0°` |
| `30°` | `约 1.5–1.6°` |
| `60°` | `约 3.0°` |
| `90°` | `约 3.8–4.0°` |

上述表格只用于验证“随舵机角增大而逐渐饱和”的趋势。量角器分辨率、视差、轨道测量基准、中位偏置和人工拨动误差均可能影响结果。

## 4. 推测参数

左侧连接结构存在约 `5–6.5 cm` 的高度或连杆相关尺寸，但当前尚未明确它究竟表示：

- 舵机轴到顶板的高度；
- 连杆两铰点中心距；
- 轨道连接点到某一基准面的高度；
- 或上述尺寸的组合。

在重新定义测量端点之前，不应把该数值代入精确闭环运动学模型。

## 5. 一级运动学近似模型

暂时把舵机摇臂端点的有效垂直位移近似为：

```text
h(α) = r_servo · sin(α)
```

轨道绕右侧固定端转动，进一步近似为：

```text
θ(α) = atan(h(α) / L_drive)
     = atan(r_servo · sin(α) / L_drive)
```

代入当前粗略参数：

```text
r_servo = 1.5 cm
L_drive = 24.8 cm
θ(α) ≈ atan(1.5 · sin(α) / 24.8)
```

三角函数内部必须使用弧度。程序接口可以接收角度制，但进入 `sin()`、`atan()` 前必须转换为弧度，输出轨道角时再转换回角度。

该近似模型的趋势预测约为：

| 舵机角 | 模型轨道角 |
| ---: | ---: |
| `30°` | `约 1.73°` |
| `60°` | `约 3.00°` |
| `90°` | `约 3.46°` |

这只是一级近似模型。它忽略连杆长度约束、两端水平/垂直偏置、连杆倾角、装配间隙、零位偏置、正负方向不对称以及舵机实际角度误差。

## 6. 钢珠动力学初模

若钢珠可近似为在轨道上纯滚动的均匀实心球，经典模型为：

```text
x_ddot = (5/7) · g · sin(θ)
```

小角度时：

```text
x_ddot ≈ (5/7) · g · θ
```

与一级运动学模型组合后：

```text
x_ddot ≈ (5/7) · g · sin(θ(α))
```

这里 `x` 是钢珠沿轨道方向的位置，`α` 是舵机相对中位角，`θ` 是轨道倾角。该模型当前没有包含静摩擦、滚动阻力、粘性阻尼、轨道零位偏置、车体运动引起的外部加速度、舵机延迟、死区和回差，因此暂不用于直接设计 PID、LQR 或状态空间控制器。

## 7. 尚未验证的假设

- 舵机命令角等于舵机摇臂实际角度。
- 舵机中位对应轨道真实水平位置。
- 舵机正负方向具有相同运动学关系。
- 连杆回差和结构弹性可以忽略。
- 钢珠始终纯滚动且不打滑。
- 轨道表面沿全长具有相同摩擦特性。
- `r_servo=1.5 cm` 和 `L_drive=24.8 cm` 的测量基准完全正确。
- 轨道角与舵机角只需要单值函数描述，不需要区分正向和反向行程。
- 小车运动产生的纵向加速度可以忽略。

## 8. 视觉坐标约定

坐标必须按机械功能定义，不能按相机画面的左右定义：

- Marker A：右侧固定转轴端，`position=-1`。
- Marker B：左侧舵机驱动端，`position=+1`。
- 两端中点：`position=0`。

相机镜像、翻转或安装方向变化后，A 可能出现在画面左侧或右侧，但它仍然是固定端。钢珠位置应投影到有向向量 `A -> B`，而不是直接使用图像 `x` 坐标。

初期相机视野应尽量覆盖整条有效轨道，因为大倾角时钢珠可能快速滚向端部。当前优先验证钢珠检测结果是否连续、单调，不发送舵机控制量。

## 9. 标准化标定计划

拿到下位机或舵机测试器后，建议采样：

```text
-90, -75, -60, -45, -30, -20, -10, 0,
 10,  20,  30,  45,  60,  75, 90 deg
```

每个角度至少分别从较小命令向较大命令到达一次（`forward`），再从较大命令向较小命令到达一次（`reverse`），记录：

- 方向；
- 舵机命令角；
- PWM 脉宽；
- 实际轨道倾角；
- 测量方法；
- 光照、负载、停留时间和异常现象。

轨道倾角优先使用手机水平仪、固定侧视相机直线拟合或经过校准的 IMU。标定时必须固定零位定义、正负方向、测量位置和等待时间。

后续控制器应优先使用实测标定表或由实测数据拟合的模型；理论公式只用于趋势检查、初值估计和发现异常数据。

## 10. PC 建模工具

工具位置：`tools/mechanical_model.py`。它只在电脑端运行，可以使用 NumPy、pandas 和 matplotlib，不得上传到 K230。

电脑端依赖：

```powershell
python -m pip install numpy pandas matplotlib
```

该命令只在电脑 A/B 的普通 Python 环境中执行，不在 K230 或 CanMV IDE 中执行。

示例：

```powershell
python tools/mechanical_model.py --angles 0 30 60 90
python tools/mechanical_model.py --csv data/servo_rail_calibration.csv
python tools/mechanical_model.py --csv data/servo_rail_calibration.csv --poly-degree 2 --plot-output logs/servo_rail.png --no-show
```

CSV 中没有有效 `rail_angle_deg` 数据时，工具只输出和绘制理论曲线；有足够实测点时，才拟合线性、带偏置正弦比例和低阶多项式模型。拟合成功只说明模型对当前样本的数值误差较小，不等于机构模型已经得到物理验证。
